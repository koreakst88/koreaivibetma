## Шаг 1: Обзор решений для хранения данных (10 мин)

### Часть 1: Проблема — куда сохранять данные?

Представьте: пользователь добавил 3 пиццы в корзину. Потом:

- Закрыл приложение
- Или обновил страницу
- Или перезапустил Telegram

**Что произойдёт с корзиной?**

Если данные только в памяти (JavaScript-переменные):
- Корзина обнулится. Всё пропадёт.

**Почему?**
Потому что переменные JavaScript живут только пока страница открыта. Закрыли страницу → память очистилась → данные исчезли.

**Решение:**
Нужно сохранять данные где-то постоянно. Чтобы при перезагрузке страницы корзина оставалась.

---

### Часть 2: Варианты хранения данных

Есть несколько способов хранить данные. Разберём каждый.

#### Вариант 1: LocalStorage (простое хранилище в браузере)

**Что это:**
LocalStorage — это встроенное хранилище браузера. Думайте о нём как о блокноте, который браузер хранит на вашем компьютере. Данные записываются в виде текста (пары: ключ → значение).

**Плюсы:**
- ✅ Очень просто использовать — всего 2 команды: сохранить и прочитать
- ✅ Не нужен сервер — всё хранится локально
- ✅ Бесплатно — встроено в браузер
- ✅ Данные не пропадают — хранятся до тех пор, пока пользователь не очистит кэш
- ✅ Идеально для MVP — быстро запустить и протестировать

**Минусы:**
- ⚠️ Только на одном устройстве — если пользователь откроет приложение на другом телефоне, корзина будет пустая
- ⚠️ Небольшой объём — до 5-10 MB (для корзины этого более чем достаточно)
- ⚠️ Можно очистить — если пользователь почистит кэш браузера, данные исчезнут

**Когда использовать:**
Для MVP, корзины, настроек пользователя, временных данных. Это то, что мы будем использовать сегодня!

---

#### Вариант 2: IndexedDB (продвинутое хранилище)

**Что это:**
IndexedDB — это база данных прямо в браузере. Более мощная, чем LocalStorage. Можно хранить сложные структуры данных, делать запросы, индексы.

**Плюсы:**
- ✅ Больше объём — до 50 MB и больше
- ✅ Более гибкая — можно делать сложные запросы
- ✅ Асинхронная — не блокирует интерфейс

**Минусы:**
- ⚠️ Сложнее в использовании — нужно писать больше кода
- ⚠️ Для новичков избыточна — LocalStorage проще

**Когда использовать:**
Когда LocalStorage не хватает (очень большие объёмы данных). Для MVP — не нужна.

---

#### Вариант 3: Supabase / Firebase (облачная база данных)

**Что это:**
Supabase и Firebase — это сервисы облачных баз данных. Данные хранятся на сервере в интернете, а не на устройстве пользователя.

**Плюсы:**
- ✅ Синхронизация между устройствами — пользователь открывает на телефоне → видит свою корзину, открывает на компьютере → та же корзина
- ✅ Надёжно — данные не пропадут, даже если пользователь очистит кэш
- ✅ Можно делиться — несколько пользователей видят одни данные
- ✅ Можно делать сложные запросы — фильтры, сортировки, агрегации

**Минусы:**
- ⚠️ Сложнее настроить — нужно создать аккаунт, настроить таблицы, подключить к проекту
- ⚠️ Нужен интернет — без интернета не работает
- ⚠️ Для MVP избыточно — можно начать с LocalStorage

**Когда использовать:**
Когда нужна синхронизация между устройствами, когда пользователей много, когда нужна история заказов. Мы упомянем Supabase позже, но для MVP начнём с LocalStorage.

---

### Часть 3: Что мы выбираем и почему

Для нашего приложения доставки пиццы выбираем **LocalStorage**.

**Почему:**
- ✅ Простота — 2 команды, и всё работает
- ✅ Скорость разработки — не нужно настраивать сервер
- ✅ Достаточно для MVP — корзина, заказы — всё влезет
- ✅ Бесплатно — никаких подписок

**Как это работает:**

Представьте шкаф с ящиками. Каждый ящик — это ключ (название). В ящике лежат данные (значение).

**Пример:**
Ящик с названием `cart` (корзина). Внутри лежит список пицц: `["Маргарита", "Пепперони"]`.

**Команды:**

**Сохранить данные:**
```javascript
localStorage.setItem('cart', JSON.stringify(cartArray));
```

**Прочитать данные:**
```javascript
const cartArray = JSON.parse(localStorage.getItem('cart'));
```

Не пугайтесь `JSON.stringify` и `JSON.parse`. Это просто преобразование:
- `JSON.stringify` → превращает массив в текст (чтобы сохранить)
- `JSON.parse` → превращает текст обратно в массив (чтобы использовать)

LocalStorage хранит только текст, поэтому нужно преобразовывать.
