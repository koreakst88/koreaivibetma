## Шаг 1: Что такое автоматизация + Обзор инструментов (15 мин)

### Часть 1: Что такое автоматизация?

**Автоматизация** — это когда вы настраиваете сценарий: если произошло А, сделай Б.

**Примеры из жизни:**

Представьте:
- Если дверь открылась → включить свет
- Если температура ниже 18° → включить обогреватель
- Если пришло письмо → переслать на другой адрес

Это автоматизация в реальном мире.

**В цифровом мире:**
То же самое, но с приложениями и сервисами:
- Если новая строка в Google Sheets → отправить уведомление в Telegram
- Если новый email → сохранить вложение в Dropbox
- Если новый заказ в приложении → записать в базу данных

Вы настраиваете один раз, и дальше всё работает автоматически!

**Зачем это нужно?**
- ✅ Экономия времени — не нужно делать рутину вручную
- ✅ Меньше ошибок — компьютер не забывает
- ✅ Масштабирование — можно обработать 1000 заказов так же легко, как 1
- ✅ Фокус на важном — рутина работает сама, вы занимаетесь бизнесом

Для нашего приложения доставки пиццы автоматизация означает:
> Пользователь оформил заказ → вам автоматически пришло уведомление в Telegram.
> Не нужно каждые 5 минут проверять приложение!

### Часть 3: Обзор инструментов автоматизации

Есть несколько популярных платформ для автоматизации без кода:

#### 1. Make (бывший Integromat) ⭐ — мой выбор для курса

**Что это:**
- Визуальный конструктор автоматизации
- Drag-and-drop интерфейс (перетаскивание блоков)
- Очень мощный и гибкий

**Плюсы:**
- ✅ Бесплатно до 1000 операций/месяц — достаточно для старта
- ✅ Визуальный редактор — видишь всю схему сразу
- ✅ Много интеграций — 1500+ сервисов
- ✅ Мощные функции — фильтры, роутеры, итераторы
- ✅ Поддержка вебхуков — можем подключить Mini App

**Минусы:**
- ⚠️ Чуть сложнее для новичка (но мы разберёмся!)

**Цена:**
- Free: 1000 операций/месяц
- Pro: $9/мес — 10,000 операций

---

#### 2. Zapier

**Что это:**
- Самая популярная платформа автоматизации
- Простой интерфейс

**Плюсы:**
- ✅ Очень простой — самый лёгкий для новичков
- ✅ Огромная библиотека — 5000+ приложений
- ✅ Готовые шаблоны — можно взять готовое решение

**Минусы:**
- ⚠️ Дорого — бесплатно только 100 задач/месяц
- ⚠️ Меньше гибкости — чем Make

**Цена:**
- Free: 100 задач/месяц (мало!)
- Starter: $20/мес — 750 задач

---

#### 3. n8n

**Что это:**
- Open-source автоматизация
- Можно установить на свой сервер

**Плюсы:**
- ✅ Бесплатно — если разворачиваете сами
- ✅ Полный контроль — данные на вашем сервере
- ✅ Гибкость — как Make, но бесплатнее

**Минусы:**
- ⚠️ Нужен сервер — технически сложнее
- ⚠️ Настройка — требует времени

---

#### 4. Pabbly Connect

**Что это:**
- Альтернатива Zapier/Make

**Плюсы:**
- ✅ Дешевле — $19/мес за неограниченные операции
- ✅ Без лимитов на операции (только на задачи)

**Минусы:**
- ⚠️ Меньше популярен — меньше гайдов

---

| Инструмент | Бесплатно | Сложность | Гибкость | Рекомендация |

### Часть 4: Основные понятия Make

Прежде чем начать, разберём базовые понятия Make:

**1. Scenario (Сценарий)**
- Это ваша автоматизация
- Набор модулей, связанных между собой
- Пример: «Новая строка в таблице → отправить в Telegram»

**2. Module (Модуль)**
- Это один шаг в сценарии
- Каждый модуль — это действие в одном сервисе
- Примеры модулей:
  - Google Sheets: «Watch Rows» (следить за новыми строками)
  - Telegram: «Send a Message» (отправить сообщение)
  - HTTP: «Make a Request» (сделать запрос)

**3. Trigger (Триггер)**
- Первый модуль в сценарии
- Событие, которое запускает автоматизацию
- Примеры:
  - Новая строка в таблице
  - Новый файл в папке
  - Webhook (входящий запрос)

**4. Action (Действие)**
- Все модули после триггера
- Что делать, когда триггер сработал
- Примеры:
  - Отправить сообщение
  - Создать запись
  - Отправить email

**5. Data Flow (Поток данных)**
- Данные передаются от модуля к модулю
- Каждый модуль получает данные от предыдущего
- Пример:
  - Триггер: новая строка (данные: имя, телефон, адрес)
  - Действие: отправить в Telegram (использует данные: имя, адрес)

**6. Operation (Операция)**
- Одно выполнение модуля = 1 операция
- Бесплатно: 1000 операций/месяц
- Пример: 100 заказов × 2 модуля = 200 операций

Это основные термины. Сейчас увидите их в действии!

### Часть 3: Подключение Mini App к Google Sheets

Теперь настроим Mini App, чтобы заказы автоматически записывались в Google Sheets.

#### Подготовка: Создать Google Apps Script

Google Apps Script — это способ записывать данные в таблицу через код (скрипт). Мы создадим endpoint (URL), куда Mini App будет отправлять данные.

---

#### Шаг 1: Открыть редактор скриптов

1. В вашей таблице **Pizza Orders**
2. Меню: **Extensions** (Расширения) → **Apps Script**
3. Откроется редактор кода

---

#### Шаг 2: Вставить код скрипта

Удалите весь код в редакторе и вставьте:

```javascript
function doPost(e) {
  try {
    // Получаем данные из запроса
    var data = JSON.parse(e.postData.contents);

    // Открываем таблицу
    var sheet = SpreadsheetApp.getActiveSpreadsheet().getActiveSheet();

    // Добавляем новую строку с данными заказа
    sheet.appendRow([
      data.orderId,
      data.date,
      data.name,
      data.phone,
      data.address,
      data.items,
      data.total
    ]);

    // Возвращаем успешный ответ
    return ContentService.createTextOutput(JSON.stringify({
      'status': 'success',
      'message': 'Order saved'
    })).setMimeType(ContentService.MimeType.JSON);

  } catch (error) {
    // Если ошибка — возвращаем её
    return ContentService.createTextOutput(JSON.stringify({
      'status': 'error',
      'message': error.toString()
    })).setMimeType(ContentService.MimeType.JSON);
  }
}
```

---

#### Шаг 3: Деплой скрипта

1. Нажмите **Deploy** (Развернуть) → **New deployment** (Новое развёртывание)
2. Выберите тип: **Web app** (Веб-приложение)
3. Настройки:
   - **Description:** «Pizza Orders API»
   - **Execute as:** «Me» (От моего имени)
   - **Who has access:** «Anyone» (Любой пользователь)
4. Нажмите **Deploy**
5. Скопируйте **Web app URL** (длинная ссылка)
   - Выглядит примерно так:
   ```
   https://script.google.com/macros/s/XXXXX.../exec
   ```

Эта ссылка — ваш endpoint. Mini App будет отправлять сюда данные заказа.

```
     items: cart.map(item => `${item.name} x${item.quantity}`).join(', '),
     total: calculateTotal() + '₽'
   };

2. Отправить POST-запрос на Apps Script:
   const APPS_SCRIPT_URL = 'ВАШ_URL_ИЗ_APPS_SCRIPT';

   fetch(APPS_SCRIPT_URL, {
     method: 'POST',
     mode: 'no-cors', // важно для Apps Script
     headers: { 'Content-Type': 'application/json' },
     body: JSON.stringify(orderData)
   });

3. Не блокировать переход на success.html:
   - Отправить запрос асинхронно
   - Сразу перейти на success.html (не ждать ответа)

4. Для безопасности:

   - Закомментировать ссылку для будущего изменения

```

## Шаг 3: Другие интеграции + Монетизация автоматизации (15 мин)

### Часть 1: Интеграции с популярными сервисами

#### Google Sheets — центр автоматизации

Google Sheets — это универсальная база данных для автоматизации.

**Почему?**
- ✅ Бесплатно — неограниченное количество таблиц
- ✅ Доступно везде — браузер, телефон, API
- ✅ Просто — как Excel, но онлайн
- ✅ Интеграции — почти все сервисы работают с Sheets

**Популярные сценарии с Google Sheets:**

**Сбор лидов:**
- Форма на сайте → запись в Sheets → email клиенту

**Управление контентом:**
- Sheets с расписанием постов → автопостинг в соцсети

**Учёт финансов:**
- Sheets с расходами → автоотчёты раз в неделю

**CRM для малого бизнеса:**
- Sheets как база клиентов → автоматические напоминания

Если не знаете, с чего начать автоматизацию — начните с Sheets!

---

#### Telegram — универсальный интерфейс

Telegram — идеальный канал для уведомлений и управления.

**Популярные сценарии с Telegram:**

**Уведомления (как у нас):**
- Новый заказ → Telegram
- Новый лид → Telegram
- Ошибка на сайте → Telegram

**Управление через бота:**
- Отправить команду боту → запустить процесс
- Пример: «Отправь отчёт» → бот формирует и отправляет отчёт

**Групповые уведомления:**
- Новая заявка → уведомление в групповой чат команды

**Интерактивные боты:**
- Бот с кнопками → выбор действия → автоматизация

Telegram + Make = мощная связка для бизнеса!

#### Email — автоматические рассылки

Email остаётся важным каналом коммуникации.

**Популярные сценарии с Email:**

**Автоответы:**
- Новый заказ → email клиенту с подтверждением

**Рассылки:**
- Новый подписчик в Sheets → добавить в Mailchimp → приветственная серия

**Уведомления команде:**
- Новая заявка → email менеджеру

**Отчёты:**
- Раз в неделю → собрать данные → отправить PDF-отчёт

**Сервисы для работы с email в Make:**
- Gmail (бесплатно)
- Mailchimp (email-маркетинг)
- SendGrid (транзакционные письма)

---

#### Airtable — альтернатива Google Sheets

Airtable — это Google Sheets на стероидах.

**Почему Airtable:**
- ✅ Больше типов полей — файлы, ссылки, чекбоксы, рейтинги
- ✅ Визуализация — канбан, календарь, галерея
- ✅ Автоматизация встроена — есть свои сценарии
- ✅ API — легко интегрируется

**Минусы:**
- ⚠️ Бесплатно только 1,000 записей (Sheets — неограниченно)

**Когда использовать Airtable вместо Sheets:**
- Нужны сложные типы данных (файлы, вложения)
- Нужна визуализация (канбан, календарь)
- Работа с командой (комментарии, упоминания)

Для простых задач — Sheets. Для сложных — Airtable.

**6. Автоматические напоминания:**
- Sheets с задачами → Make → каждое утро → Telegram с напоминанием
