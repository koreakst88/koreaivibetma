## Шаг 1: Платежи в Telegram + Уведомления через бота (25 мин)

### Часть 1: Зачем нужны платежи?

Если вы создаёте Mini App для бизнеса, рано или поздно встанет вопрос: как принимать оплату?

**Варианты:**

1. Пользователь оформляет заказ → оператор звонит → оплата при получении (как сейчас)
2. Пользователь оформляет заказ → оплата онлайн прямо в приложении

**Второй вариант удобнее:**
- ✅ Не нужно обрабатывать каждый заказ вручную
- ✅ Деньги приходят сразу
- ✅ Меньше отказов (пользователь уже оплатил)
- ✅ Автоматизация процесса

Telegram предлагает несколько способов приёма платежей. Сейчас разберём их обзорно — без глубокой реализации (это тема для отдельного курса).

### Часть 2: Варианты оплаты в Telegram

Telegram поддерживает несколько платёжных систем:

#### Вариант 1: Telegram Stars (⭐ встроенная валюта)

**Что это:**
Telegram Stars — это внутренняя валюта Telegram. Пользователи покупают Stars за реальные деньги, а потом тратят их в Mini Apps и ботах.

**Как это работает:**
1. Пользователь покупает Stars в Telegram (например, 100 Stars за $1)
2. Ваше приложение показывает цену в Stars: «Пицца Маргарита — 50 ⭐»
3. Пользователь оплачивает Stars
4. Telegram переводит вам деньги (за вычетом комиссии)

**Плюсы:**
- ✅ Встроено в Telegram — не нужны сторонние сервисы
- ✅ Пользователи доверяют Telegram
- ✅ Простая интеграция
- ✅ Быстрые платежи

**Минусы:**
- ⚠️ Комиссия Telegram (обычно 30%)
- ⚠️ Пользователи сначала покупают Stars (дополнительный шаг)
- ⚠️ Не все пользователи знают про Stars

**Когда использовать:**
Для цифровых товаров, донатов, небольших покупок.

---

#### Вариант 2: TON (криптовалюта Telegram)

**Что это:**
TON (The Open Network) — это блокчейн-платформа, связанная с Telegram. Пользователи могут платить в криптовалюте TON.

**Как это работает:**
1. Пользователь имеет TON-кошелёк
2. Ваше приложение показывает цену в TON: «Пицца — 5 TON»
3. Пользователь отправляет TON на ваш кошелёк
4. Транзакция мгновенная, комиссии низкие

**Плюсы:**
- ✅ Низкие комиссии (1-2%)
- ✅ Мгновенные транзакции
- ✅ Полный контроль над деньгами
- ✅ Модно для крипто-аудитории

**Минусы:**
- ⚠️ Не все пользователи имеют криптокошельки
- ⚠️ Волатильность курса (цена TON меняется)
- ⚠️ Сложнее интегрировать

**Когда использовать:**
Для крипто-проектов, NFT, Web3-приложений.

---

#### Вариант 3: Telegram Payments (интеграция со Stripe, PayPal и др.)

**Что это:**
Telegram позволяет интегрировать классические платёжные системы:
- Stripe (карты)
- PayPal
- Яндекс.Касса
- ЮKassa
- И другие

**Как это работает:**
1. Ваше приложение вызывает метод `Telegram.WebApp.invoiceOpened()`
2. Открывается платёжная форма Telegram
3. Пользователь вводит данные карты
4. Оплата проходит через выбранную систему
5. Вам приходят деньги (за вычетом комиссии платёжной системы)

**Плюсы:**
- ✅ Привычные способы оплаты (карты)
- ✅ Широкая аудитория (почти у всех есть карты)
- ✅ Telegram обрабатывает безопасность

**Минусы:**
- ⚠️ Нужна регистрация в платёжной системе (Stripe и т.д.)
- ⚠️ Комиссии (2-5% обычно)
- ⚠️ Сложнее настроить

**Когда использовать:**
Для коммерческих приложений, интернет-магазинов, сервисов с большими чеками.

### Часть 6: Уведомления через Telegram бота

#### Зачем нужны уведомления?

Сейчас пользователь оформляет заказ, и... всё. Никто не узнаёт о новом заказе!

Нужно, чтобы:
- Вы (владелец бизнеса) получили уведомление: «Новый заказ #123»
- Оператор получил детали заказа
- Можно даже отправить подтверждение клиенту

Это делается через Telegram бота и webhook.

---

#### Как это работает — схема

Вот как работает отправка уведомлений:

```
1. Пользователь оформляет заказ в Mini App
   ↓
2. Mini App отправляет данные на ваш сервер (webhook)
   ↓
3. Сервер получает данные заказа
   ↓
4. Сервер вызывает Telegram Bot API: "Отправить сообщение"
   ↓
5. Бот отправляет сообщение вам в Telegram:

   🍕 Новый заказ #12345
   Клиент: Иван Иванов
   Телефон: +7 (912) 345-67-89
   Адрес: ул. Ленина, 10, кв. 5
   Состав: Маргарита x2, Пепперони x1
   Сумма: 1250₽
```

**Что нужно:**
- Backend (сервер) — принимает данные от Mini App
- Telegram Bot Token — для отправки сообщений
- Chat ID — куда отправлять (ваш Telegram ID)

Для MVP можно обойтись без этого (оператор проверяет LocalStorage или таблицу заказов). Но для масштабирования — уведомления критичны!

**Практическую реализацию уведомлений через Make мы разберём на Дне 6** — это позволит настроить уведомления без backend, через связку Google Sheets + Make + Telegram.

  // Отправляем сообщение админу

## Шаг 2: Работа с внешними API — примеры (10 мин)

### Часть 1: Что такое API?

**API** — это Application Programming Interface (интерфейс для взаимодействия программ).

**Простыми словами:**
API — это способ для вашего приложения попросить другой сервис что-то сделать.

**Примеры из жизни:**

Представьте ресторан:
- **Вы (ваше приложение)** — клиент
- **Официант (API)** — принимает ваш заказ
- **Кухня (внешний сервис)** — готовит блюдо
- **Официант приносит вам результат**

Вы не заходите на кухню сами — вы просите официанта.

**В программировании:**

Ваше приложение отправляет запрос:
> «Дай мне координаты адреса: ул. Ленина, 10»

API возвращает ответ:
```json
{ "lat": 55.7558, "lng": 37.6173 }
```

Вы используете эти координаты (например, показываете на карте).

**Для работы с API нужно:**
1. Найти нужный API (Google Maps, OpenWeather, и т.д.)
2. Получить ключ доступа (обычно бесплатно)
3. Отправить запрос из вашего приложения
4. Получить ответ и использовать данные

- Построить маршрут для курьера

**Как реализовать:**

Встроить Google Maps или Yandex Maps:
```html
<iframe
  src="https://www.google.com/maps/embed/v1/place?key=API_KEY&q=адрес"
  width="100%" height="300">
</iframe>
```

**Промпт для ИИ:**
```
Добавь карту с адресом доставки на странице подтверждения заказа.

1. На success.html показать встроенную карту Google Maps
2. Отметить точку: адрес из заказа
3. Размер карты: 100% ширина, 300px высота
4. Под картой: кнопка "Открыть в Google Maps" (внешняя ссылка)

Используй Google Maps Embed API.
API Key: [ваш ключ]
```

### Часть 3: Где найти API и как получить ключи

**Где искать API для ваших задач:**

**Популярные источники:**

1. **RapidAPI** (https://rapidapi.com)
   - Тысячи API в одном месте
   - Удобный поиск
   - Единый ключ для всех API

2. **Google Cloud Platform**
   - Maps API, Geocoding, Distance Matrix
   - Бесплатно до определённых лимитов

3. **OpenWeatherMap** (погода)
4. **ExchangeRate-API** (валюты)
5. **Stripe/PayPal** (платежи)
6. **Twilio** (SMS)

**Как получить API ключ:**
1. Зарегистрироваться на сайте API
2. Создать проект/приложение
3. Получить API Key (длинная строка символов)
4. Использовать в коде

Обычно есть бесплатный тариф (лимит запросов в месяц).

Когда нужен API — просто попросите ИИ:
> «Найди бесплатный API для [задача]. Покажи пример использования.»

ИИ подскажет варианты и напишет код!
