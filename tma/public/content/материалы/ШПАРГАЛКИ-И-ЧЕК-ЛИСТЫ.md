### 2. Формула промптов (4-частная)

#### Базовая формула

```
┌─────────────────────────────────────────────────────────┐
│              ХОРОШИЙ ПРОМПТ =                           │
│                                                         │
│  1. ЧТО создать                                         │
│  2. КАКИЕ элементы включить                             │
│  3. КАКОЕ поведение                                     │
│  4. КАКОЙ стиль/дизайн                                  │
└─────────────────────────────────────────────────────────┘
```

---

#### Детальная структура

**1. ЧТО нужно создать**
```
Начни с общей задачи:
- "Создай форму регистрации"
- "Создай карточку товара"
- "Создай страницу с каталогом"
```

**2. КАКИЕ элементы включить**
```
Перечисли конкретные элементы:
- Поле для ввода имени
- Поле для ввода email
- Кнопка "Отправить"
- Чекбокс согласия
```

**3. КАКОЕ поведение**
```
Опиши, что происходит при действиях:
- При нажатии кнопки → проверить поля
- Если заполнены → показать "Успешно!"
- Если нет → показать ошибку
```

**4. КАКОЙ стиль/дизайн**
```
Опиши визуальные детали:
- Цветовая схема: синий и белый
- Кнопки с закруглёнными углами
- Современный, минималистичный дизайн
```

---

#### Пример хорошего промпта

```
Создай форму обратной связи.

Элементы:
- Поле для ввода имени (обязательное)
- Поле для ввода email (обязательное)
- Текстовое поле для сообщения (5 строк)
- Кнопка "Отправить"

Поведение:
- При нажатии "Отправить" → проверить заполнение полей
- Если все поля заполнены → показать "Сообщение отправлено!"
- Если есть пустые поля → подсветить их красным

Дизайн:
- Современный стиль
- Цвета: синий (#007BFF) для кнопки, белый фон
- Форма по центру страницы
- Поля ввода с рамкой серого цвета
```

---

#### Шаблон для копирования

```markdown
[ДЕЙСТВИЕ]: Создай [ЧТО]

Элементы:
- [элемент 1]
- [элемент 2]
- [элемент 3]

Поведение:
- При [действие] → [реакция]
- Если [условие] → [результат]

Дизайн:
- [цвет/стиль 1]
- [цвет/стиль 2]
- [дополнительные требования]
```

### 4. Git команды (основные)

#### Базовые команды

```bash
# Инициализация репозитория
git init

# Проверка статуса (какие файлы изменены)
git status

# Добавление всех файлов в индекс
git add .

# Добавление конкретного файла
git add имя_файла.txt

# Создание коммита
git commit -m "Описание изменений"

# Отправка изменений на GitHub
git push

# Получение изменений с GitHub
git pull
```

---

#### Работа с ветками

```bash
# Показать все ветки
git branch

# Создать новую ветку
git branch имя_ветки

# Переключиться на ветку
git checkout имя_ветки

# Создать и переключиться
git checkout -b имя_ветки

# Слить ветки
git merge имя_ветки
```

---

#### Отмена изменений

```bash
# Отменить изменения в файле (до git add)
git checkout -- имя_файла

# Удалить файл из индекса (после git add)
git reset HEAD имя_файла

# Отменить последний коммит (сохранить изменения)
git reset --soft HEAD~1

# Отменить коммит (удалить изменения)
git reset --hard HEAD~1
```

---

#### Просмотр истории

```bash
# Показать историю коммитов
git log

# Краткая история
git log --oneline

# История с изменениями
git log --stat

# Показать изменения в последнем коммите
git show
```

---

#### Частые сценарии

**Первый коммит проекта:**
```bash
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/username/repo.git
git push -u origin main
```

**Ежедневная работа:**
```bash
git add .
git commit -m "Описание изменений"
git push
```

**Обновление из репозитория:**
```bash
git pull origin main
```

### 5. Troubleshooting (частые ошибки и решения)

#### Ошибка: "git: command not found"

**Проблема:** Git не установлен или не в PATH

**Решение:**
```bash
# Windows: установить Git с https://git-scm.com/download/win
# macOS: установить через Homebrew
brew install git

# Или проверить путь
which git  # macOS
where git  # Windows
```

---

#### Ошибка: "Permission denied (publickey)"

**Проблема:** SSH ключ не настроен

**Решение:**
```bash
# Создать SSH ключ
ssh-keygen -t ed25519 -C "your_email@example.com"

# Добавить ключ в SSH agent
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519

# Добавить ключ в GitHub (Settings → SSH and GPG keys)
cat ~/.ssh/id_ed25519.pub
```

---

#### Ошибка: "Failed to connect to GitHub"

**Проблема:** Проблемы с сетью или firewall

**Решение:**
```bash
# Проверить подключение
ping github.com

# Использовать HTTPS вместо SSH
git remote set-url origin https://github.com/username/repo.git

# Или настроить прокси
git config --global http.proxy http://proxyuser:proxypwd@proxy.server.com:port
```

---

#### Ошибка: "Changes not staged for commit"

**Проблема:** Файлы не добавлены в индекс

**Решение:**
```bash
# Добавить все изменённые файлы
git add .

# Или конкретные файлы
git add файл1.txt файл2.txt

# Проверить статус
git status
```

---

#### Ошибка: "Merge conflict"

**Проблема:** Конфликт слияния

**Решение:**
```bash
# Найти файлы с конфликтами
git status

# Открыть файлы, найти маркеры конфликтов:
# <<<<<<< HEAD
# ваши изменения
# =======
# изменения из remote
# >>>>>>>

# Исправить конфликты вручную
# Удалить маркеры <<<<<<<, =======, >>>>>>>

# Добавить исправленные файлы
git add имя_файла

# Завершить слияние
git commit -m "Resolve merge conflict"
```

---
