## Шаг 1: Аналитика в Mini Apps (15 мин)

### Часть 1: Зачем нужна аналитика?

**Аналитика** — это данные о том, как пользователи взаимодействуют с вашим приложением.

**Что можно узнать:**

**📊 Количество пользователей:**
- Сколько человек открыло приложение сегодня/неделю/месяц?
- Сколько новых пользователей?
- Сколько вернулись повторно?

**📊 Поведение пользователей:**
- Какие страницы просматривают?
- Где больше всего времени проводят?
- На каком этапе уходят (не завершают заказ)?

**📊 Конверсии:**
- Сколько пользователей добавили товар в корзину?
- Сколько оформили заказ?
- Какая конверсия (процент оформивших заказ от общего числа)?

**📊 Популярные товары:**
- Какие пиццы заказывают чаще всего?
- Какие игнорируют?

---

**Зачем это знать?**
- ✅ Принимать решения на основе данных — не гадать, а знать
- ✅ Улучшать приложение — убрать то, что не работает
- ✅ Увеличивать продажи — оптимизировать воронку
- ✅ Понимать аудиторию — кто ваши пользователи

Без аналитики вы летите вслепую. С аналитикой — видите полную картину!

### Часть 2: Варианты аналитики для Mini Apps

Есть несколько популярных инструментов:

#### 1. Google Analytics (GA4) ⭐

**Что это:**
- Самый популярный сервис веб-аналитики
- Бесплатный
- Огромные возможности

**Плюсы:**
- ✅ Бесплатно навсегда
- ✅ Мощный — отчёты, воронки, сегментация
- ✅ Интеграция — с Google Ads, Search Console и др.
- ✅ Стандарт индустрии — все знают GA

**Минусы:**
- ⚠️ Сложноват для новичка (но мы упростим!)

**Когда использовать:**
- Для серьёзных проектов
- Когда нужна глубокая аналитика

---

#### 2. Яндекс.Метрика

**Что это:**
- Российский аналог Google Analytics
- Бесплатный
- Популярен в СНГ

**Плюсы:**
- ✅ Бесплатно
- ✅ Вебвизор — записи сессий пользователей (видите, как они кликают)
- ✅ Карты кликов — видите, куда кликают пользователи
- ✅ Простой интерфейс — проще, чем GA

**Минусы:**
- ⚠️ Меньше функций, чем GA

**Когда использовать:**
- Для проектов в СНГ
- Когда нужен вебвизор

---

#### 3. Простая аналитика через Make

**Что это:**
- Собственная система на основе Google Sheets
- Каждое событие → строка в таблице

**Плюсы:**
- ✅ Полный контроль — вы владеете данными
- ✅ Гибкость — можете отслеживать что угодно
- ✅ Бесплатно

**Минусы:**
- ⚠️ Нужно настраивать вручную
- ⚠️ Нет готовых отчётов (придётся строить самим)

**Когда использовать:**
- Для MVP
- Когда нужны специфичные метрики

Для курса мы подключим Google Analytics — это стандарт.

### Часть 3: Подключение Google Analytics

Подключим GA к приложению через Cline.

#### Шаг 1: Создать аккаунт Google Analytics

1. Перейдите на: `analytics.google.com`
2. Нажмите **Начать измерения** (или Start measuring)
3. Создайте аккаунт:
   - Название аккаунта: «Pizza Delivery App» (любое)
4. Создайте ресурс (Property):
   - Название ресурса: «Pizza Delivery Mini App»
   - Часовой пояс: ваш
   - Валюта: RUB (или ваша)
5. Выберите категорию: «Food & Drink» (Еда и напитки)
6. Цели бизнеса: можете пропустить (Next)
7. Выберите платформу: **Веб**

---

#### Шаг 2: Получить Measurement ID

1. После создания вы попадёте в настройки потока данных
2. Выберите **Веб-поток** (Web stream)
3. Введите URL вашего приложения: `https://pizza-delivery-ваш-username.vercel.app`
4. Название потока: «Pizza Delivery Production»
5. Нажмите **Создать поток** (Create stream)

Вы увидите **Measurement ID** — выглядит как: `G-XXXXXXXXXX`

**Скопируйте этот ID** — он понадобится

---

#### Шаг 3: Добавить GA в Mini App через Cline

**Промпт для Cline:**

```
Подключи Google Analytics 4 (GA4) к приложению.

ЗАДАЧА:

1. В каждом HTML-файле (index.html, cart.html, order.html, success.html):

   Добавить в <head> перед закрывающим тегом:

   <!-- Google Analytics -->
   <script async src="https://www.googletagmanager.com/gtag/js?id=G-XXXXXXXXXX"></script>
   <script>
     window.dataLayer = window.dataLayer || [];
     function gtag(){dataLayer.push(arguments);}
     gtag('js', new Date());
     gtag('config', 'G-XXXXXXXXXX');
   </script>

   (Замени G-XXXXXXXXXX на реальный Measurement ID)

2. В src/script.js добавить отслеживание событий:

   // Событие: Добавление в корзину
   function trackAddToCart(pizzaName, price) {
     gtag('event', 'add_to_cart', {
       'currency': 'RUB',
       'value': price,
       'items': [{
         'item_name': pizzaName,
         'price': price
       }]
     });
   }

   // Событие: Просмотр корзины
   function trackViewCart(total) {
     gtag('event', 'view_cart', {
       'currency': 'RUB',
       'value': total
     });
   }

   // Событие: Начало оформления
   function trackBeginCheckout(total) {
     gtag('event', 'begin_checkout', {
       'currency': 'RUB',
       'value': total
     });
   }

   // Событие: Успешная покупка
   function trackPurchase(orderId, total) {
     gtag('event', 'purchase', {
       'transaction_id': orderId,
       'currency': 'RUB',
       'value': total
     });
   }

3. Вызывать эти функции в нужных местах:
   - При клике "Добавить в корзину" → trackAddToCart()
   - При загрузке cart.html → trackViewCart()
   - При загрузке order.html → trackBeginCheckout()
   - При успешном заказе на success.html → trackPurchase()

Все изменения в HTML-файлах и src/script.js.
Комментарии на русском языке.
```

Замените `G-XXXXXXXXXX` на ваш реальный Measurement ID. Отправьте Cline.

Cline подключит Google Analytics и добавит отслеживание событий. Подождите 2-3 минуты.

### Часть 4: Основные отчёты Google Analytics

Покажу, какие отчёты смотреть в GA:

#### 1. Отчёт «Реал-тайм» (Realtime)

- Сколько пользователей онлайн прямо сейчас
- Какие страницы просматривают
- Какие события происходят

**Используйте для:** мониторинга в реальном времени

---

#### 2. Отчёт «Пользователи» (Users)

- Общее количество пользователей за период
- Новые vs вернувшиеся
- География, устройства, браузеры

**Используйте для:** понимания аудитории

---

#### 3. Отчёт «События» (Events)

- Все события: `add_to_cart`, `purchase`, `view_cart` и т.д.
- Количество каждого события
- Конверсия между событиями

**Используйте для:** анализа поведения

---

#### 4. Отчёт «Конверсии» (Conversions)

- Ключевые события (покупки)
- Конверсия: сколько % пользователей совершили покупку

**Используйте для:** оптимизации воронки продаж

---

#### 5. Отчёт «Страницы» (Pages and screens)

- Какие страницы просматривают
- Сколько времени на каждой
- Откуда уходят

**Используйте для:** улучшения навигации

Через неделю использования вы увидите полную картину:
- Сколько пользователей
- Какие пиццы популярнее
- На каком этапе теряете клиентов
- Какая конверсия в покупку

Это позволит принимать обоснованные решения!

#### Вариант 3: Продажа шаблонов

**Суть:**

**Как это работает:**
1. Создаёте универсальный шаблон (например, «Доставка еды»)

3. Продаёте шаблон на платформах:
   - Gumroad
   - Etsy
   - Телеграм-каналы для разработчиков

**Ценник:**
- Базовый шаблон: **$50-100**
- Premium шаблон (с автоматизацией): **$150-300**

---

**Преимущества:**
- ✅ Пассивный доход (создали один раз → продаёте много раз)
- ✅ Не нужно общаться с клиентами (автоматические продажи)

**Недостатки:**
- ⚠️ Нужен маркетинг (продвижение шаблона)
